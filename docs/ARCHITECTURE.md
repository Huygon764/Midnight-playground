# Midnight Examples — Architecture & Patterns

Reference guide for building Midnight blockchain examples. Read this instead of code.

## Project Structure

Each example is a monorepo with npm workspaces:

```
examples/0X-example-name/
├── contract/          # Compact smart contract
│   ├── src/
│   │   ├── *.compact           # Compact source
│   │   ├── witnesses.ts        # Witness implementations
│   │   ├── index.ts            # Exports: namespace, witnesses, CompiledContract
│   │   └── managed/            # Generated by `compact compile` (gitignored)
│   │       └── <name>/
│   │           ├── contract/    # index.js, index.d.ts (generated TS bindings)
│   │           ├── keys/        # *.prover, *.verifier (ZK proving/verification keys)
│   │           └── zkir/        # *.zkir, *.bzkir (ZK intermediate representation)
│   ├── package.json            # @example/<name>-contract
│   └── tsconfig.build.json
├── api/               # Shared API layer (used by both CLI and web)
│   ├── src/
│   │   ├── index.ts            # Main API class (deploy, join, state$, operations)
│   │   ├── common-types.ts     # Shared types (Providers, Contract, DerivedState)
│   │   └── utils.ts            # randomBytes helper
│   └── package.json            # @example/<name>-api
├── cli/               # Node.js CLI
│   ├── src/
│   │   ├── index.ts            # Entry point
│   │   ├── cli.ts              # Interactive menu loop
│   │   ├── api.ts              # Wallet setup, provider config, contract operations
│   │   ├── common-types.ts     # CLI-specific types
│   │   └── config.ts           # Network endpoints (PreprodConfig)
│   └── package.json            # @example/<name>-cli
├── web/               # Browser UI (React + Vite)
│   ├── src/
│   │   ├── main.tsx            # Entry: globals, networkId, React render
│   │   ├── globals.ts          # Buffer + process.env polyfills
│   │   ├── App.tsx             # Full UI with on-chain calls
│   │   ├── providers.ts        # DApp Connector + Midnight provider init
│   │   └── in-memory-private-state-provider.ts
│   ├── public/
│   │   ├── keys/ → symlink     # → contract/src/managed/<name>/keys
│   │   └── zkir/ → symlink     # → contract/src/managed/<name>/zkir
│   ├── index.html              # Dark theme, inline CSS
│   ├── vite.config.ts          # wasm + topLevelAwait plugins
│   ├── .env.preprod            # VITE_NETWORK_ID=preprod
│   └── package.json            # @example/<name>-web
├── test/              # Vitest tests
└── package.json       # Root workspace config
```

## Contract Pattern

### Compact Source
```compact
pragma language_version >= 0.20;
import CompactStandardLibrary;

export ledger <field>: <ADT>;
witness <name>(): <Type>;
export circuit <name>(<args>): <ReturnType> { ... }
```

**Available ADTs:** Counter, Map, Set, HistoricMerkleTree, Boolean, Opaque

**MerkleTree depth:** Use depth 8 for examples (256 entries, compiles fast). Depth 32 takes forever to compile.

### contract/src/index.ts
```typescript
export * as ContractName from "./managed/<name>/contract/index.js";
export * from "./witnesses.js";

import { CompiledContract } from "@midnight-ntwrk/compact-js";
export const CompiledXxxContract = CompiledContract.make<...>(
  "<name>",
  ContractName.Contract<PrivateState>,
).pipe(
  CompiledContract.withWitnesses(witnesses),
  CompiledContract.withCompiledFileAssets("./compiled/<name>"),
);
```

### Witnesses
```typescript
export type XxxPrivateState = { readonly secretKey: Uint8Array; };
export const witnesses = {
  localSecretKey: ({ privateState }: WitnessContext<Ledger, PS>): [PS, Uint8Array] =>
    [privateState, privateState.secretKey],
};
```

### Build Commands
```bash
npm run compact   # compact compile src/<name>.compact src/managed/<name>
npm run build     # tsc + copy managed/ to dist/
```

## API Pattern

Shared between CLI and web. Follows `references/example-bboard/api/` pattern.

### API Class
```typescript
export class XxxAPI implements DeployedXxxAPI {
  readonly deployedContractAddress: ContractAddress;
  readonly state$: Observable<DerivedState>;

  static async deploy(providers, logger?): Promise<XxxAPI>;
  static async join(providers, contractAddress, logger?): Promise<XxxAPI>;

  // Contract operations
  async someOperation(args): Promise<void> {
    await this.deployedContract.callTx.someOperation(args);
  }
}
```

### state$ Observable
```typescript
this.state$ = providers.publicDataProvider
  .contractStateObservable(this.deployedContractAddress, { type: "latest" })
  .pipe(map((s) => ContractName.ledger(s.data)));
```

Subscribes to indexer WebSocket. Auto-updates on every on-chain state change.

### Private State Management
```typescript
private static async getPrivateState(providers): Promise<PrivateState> {
  const existing = await providers.privateStateProvider.get(stateKey);
  return existing ?? createPrivateState(utils.randomBytes(32));
}
```
First call generates random key. Subsequent calls return cached value.

### common-types.ts
```typescript
import type { ImpureCircuitId } from "@midnight-ntwrk/compact-js";

export type CircuitKeys = ImpureCircuitId<Contract<PrivateState>>;
export type Providers = MidnightProviders<CircuitKeys, PrivateStateId, PrivateState>;
export type DeployedContract = DeployedContract<Contract> | FoundContract<Contract>;
```
Use `ImpureCircuitId<>` (not `Exclude<keyof>`) — the latter resolves to `string` and breaks type inference.

## CLI Pattern

### Flow
```
Wallet Setup → Deploy/Join Contract → Actions Menu Loop
```

### Wallet Setup
Uses `wallet-sdk-*` packages: HDWallet → derive keys → ShieldedWallet + UnshieldedWallet + DustWallet → WalletFacade.

### Provider Config (Node.js)
- `NodeZkConfigProvider` — reads ZK keys from filesystem
- `levelPrivateStateProvider` — LevelDB for persistent private state
- `httpClientProofProvider` — sends proofs to localhost:6300
- `indexerPublicDataProvider` — GraphQL queries to indexer
- Manual `walletProvider` + `midnightProvider` — balance/sign/submit via WalletFacade

### CLI Script
```json
"cli": "tsx src/index.ts"
```

### tsconfig (CLI)
```json
"module": "NodeNext", "moduleResolution": "NodeNext", "types": ["node"]
```

## Web Pattern

### Flow
```
Connect Lace Wallet → Deploy/Join Contract → UI Actions
```

### Provider Config (Browser)
File: `web/src/providers.ts` — follows `references/example-bboard/bboard-ui/src/contexts/BrowserDeployedBoardManager.ts`

```typescript
const initializeProviders = async (): Promise<Providers> => {
  const connectedAPI = await connectToWallet(networkId);
  const config = await connectedAPI.getConfiguration();
  // config provides: indexerUri, indexerWsUri, proverServerUri

  return {
    privateStateProvider: inMemoryPrivateStateProvider(),
    zkConfigProvider: new FetchZkConfigProvider(window.location.origin, fetch.bind(window)),
    proofProvider: httpClientProofProvider(config.proverServerUri!, keyMaterialProvider),
    publicDataProvider: indexerPublicDataProvider(config.indexerUri, config.indexerWsUri),
    walletProvider: {
      getCoinPublicKey: () => shieldedAddresses.shieldedCoinPublicKey,
      getEncryptionPublicKey: () => shieldedAddresses.shieldedEncryptionPublicKey,
      balanceTx: async (tx) => {
        const received = await connectedAPI.balanceUnsealedTransaction(toHex(tx.serialize()));
        return Transaction.deserialize("signature", "proof", "binding", fromHex(received.tx));
      },
    },
    midnightProvider: {
      submitTx: async (tx) => {
        await connectedAPI.submitTransaction(toHex(tx.serialize()));
        return tx.identifiers()[0];
      },
    },
  };
};
```

Providers are cached — `getProviders()` returns same promise on every call.

### Wallet Connection
```typescript
const connectToWallet = (networkId): Promise<ConnectedAPI> => {
  // Poll window.midnight every 100ms for Lace extension
  // Filter for compatible API version (4.x)
  // Timeout after 1s if not found, 5s if no response
  return firstValueFrom(interval(100).pipe(
    map(() => getFirstCompatibleWallet()),
    filter(w => !!w),
    take(1),
    timeout({ first: 1_000 }),
    concatMap(api => api.connect(networkId)),
    timeout({ first: 5_000 }),
  ));
};
```

### ZK Keys Serving
Dev: symlinks in `web/public/` point to `contract/src/managed/<name>/keys|zkir`
Build: copied to `dist/keys` and `dist/zkir`

`FetchZkConfigProvider` fetches from `window.location.origin` (e.g., `/keys/registerVoter.prover`).

### Private State (Browser)
`inMemoryPrivateStateProvider` — simple `Map` in RAM. Lost on page refresh.
Same as `references/example-bboard/bboard-ui/src/in-memory-private-state-provider.ts`.

### Vite Config
Must include:
- `vite-plugin-wasm` — WASM module support
- `vite-plugin-top-level-await` — required by compact-runtime
- Custom `wasm-module-resolver` plugin — handles `@midnight-ntwrk/onchain-runtime-v2`
- `optimizeDeps.exclude` — exclude WASM modules from pre-bundling
- `build.target: "esnext"` — modern browser features

### globals.ts
```typescript
import { Buffer } from "buffer";
globalThis.process = { env: { NODE_ENV: import.meta.env.MODE } };
globalThis.Buffer = Buffer;
```

### Required npm Packages (often missing from workspace hoisting)
Add these to root `package.json` if build fails:
- `rollup` — peer dep of Vite 7
- `react-refresh` — peer dep of @vitejs/plugin-react

## Common Gotchas

**DUST tokens:** Every on-chain transaction requires DUST. After deploy, wait for DUST to regenerate before next tx. `Wallet.Transacting` error usually means no DUST.

**MerkleTree depth:** Use 8 for examples. Depth 32 = millions of ZK constraints = compile takes forever.

**Proof server:** Must be running at localhost:6300 (or whatever Lace wallet config says). Without it, proof generation fails silently.

**callTx return values:** `result.public` is `CallResultPublic & FinalizedTxData`. Circuit return values are in ZK format (`AlignedValue`), not decoded. For read-only queries, use indexer (`publicDataProvider.queryContractState`) instead of `callTx`.

**findDeployedContract hangs:** SDK's `watchForDeployTxData` polls forever if contract address doesn't exist. Always add a timeout wrapper.

**npm workspace hoisting:** Some peer deps (`rollup`, `react-refresh`) don't hoist correctly. Install them explicitly at root level.

**compact-js version:** Use 2.4.0, not 3.0.0 (doesn't exist on registry).

**Network ID:** Web needs `VITE_NETWORK_ID` env var. Set in `.env.preprod` and use `--mode preprod` for both dev and build.
